Forms

Forms are used to collect data inputted by a user. They can be used as an interface for a web application, for example, or to send data across the web.

< form > - блочный тег, т.е. занимает 100% по ширине

Атрибут action="" указывает некий адрес документа, который будет обрабатывать содержимое формы.

Атрибут method="" указывает метод отправки данных. Принимает всего два значения POST и GET. GET - метод по-умолчанию, позволяет видеть вводимые данные в URL, отправляет ограниченное число символов. POST - можно отправлять фото, видео, большой текст.

Атрибут novalidate при отправке формы, она не будет проверяться на правильность заполненности.

Атрибут autocomplete="on" По умолчанию - OFF. Сработает только если была отправленна форма. Заполняет формы после возврата на страницу. Можно прописать для отдельного инпута.

Атрибут enctype="" указывает некий тип кодирования. Работать будет при методе POST. Может принимать три значения:

"application/x-www-form-urlencoded"
По умолчанию установлено именно это значение. Если в текстовое поле вводится текст и эти данные отправляются на сервер, и если эти данные содержат кириллические буквы, то они будут заменяться на их шестьнадцатиричные значения, а пробелы будут заменяться на знаки "плюс".
"multipart/form-data"
Для отправки файлов при помощи формы. Для отправки изображения, данный атрибут обязателен. Часто используется на практике.
"text/plain"
Данные не кодируются. Пробелы заменяются знаком "плюс".

Атрибут target="" помимо прочего, может быть установлено имя айфрейма, в который будет загружаться результат обработанный сервером. Например, можно вставить значение target="iframe" и создать на странице < iframe name="iframe">?

Attribute value="something" fixes what is shown at the start.

Attribute placeholder="something" shows useful text which disappeared when the user enter something.

Attribute autofocus sets which field is given focus at the start.

Attribute required means this field must be completed.

Атрибут maxlength="" ограничивает ввод определенным числом символов.

Атрибут placeholder="" полупрозрачный текст-подсказка в поле, исчезает при вводе текста.

Атрибут required поле обязательное для заполнения. Не сработает, если в теге form бедет указан атрибут novalidate.

Атрибут readonly поле можно только прочитать.

Атрибут size="" указывает размеры поля.

Атрибут spellcheck проверяет орфографию в полях типа text и search. Может принимать три значения: TRUE - будет проверяться орфография; FALSE - орфография не будет проверяться; DEFAULT - зависит от браузера, решает браузер-подсвечивать или нет.

Атрибут pattern="" указывается шаблон ввода тескта. Например наличие симлов имейла и т.п.

Атрибут name="" указывается обязательно, и нужно для обработки данных на сервере.

Атрибут checked для чекбокса и радиобаттонс. Отмечает по умолчанию. У радиобаттонс отмечается последняя кнопка (при наличии атрибута более чем на одном варианте).

Атрибут accept="" указывается для типа file. Указывает какого типа файлы можно загружать, например видео, аудио, изображения и т.п. Можно либо, присвоить значение майнсет из таблицы типов файлов, либо следующие три значения: "audio/*", "video/*", "image/*". Также, можно дополнительно указывать майнтипы: "image/jpg, image/png", здесь загружать можно только изображения с указанным расширением.

Атрибут multiple для типа file и email, позволяет выбирать несколько файлов для загрузки.

Атрибут tabindex="" порядок табов.

В HTML существуют глобальные атрибуты ID и CLASS.

Datalist - выпадающий список вариантов заполнения поля.

Можно использовать в поле URL, email и т.д.

Атрибут wrap="" отвечает за формат переноса текста на новую строку при отправке данных на сервер. Может иметь три значения: SOFT - все в одну строку; HARD - оправится с переносами; OFF.
